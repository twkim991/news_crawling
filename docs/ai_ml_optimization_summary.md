# AI/ML 적용 기술 및 성능 최적화 개념 정리

## 1) 프로젝트에 적용된 AI/머신러닝 기술

- **문장 임베딩 기반 표현 학습**
  - `sentence-transformers`의 `intfloat/multilingual-e5-small` 모델로 기사 텍스트를 벡터화합니다.
  - 동일 임베딩 인프라를 이진 분류(tech/non-tech)와 세부 카테고리 분류에 공통 활용합니다.

- **기술 뉴스 이진 분류기 (Supervised Learning)**
  - AG News 데이터를 `Sci/Tech vs Non-Tech`로 재라벨링해 학습 데이터로 사용합니다.
  - 분류 모델은 `LogisticRegression(max_iter=2000, class_weight="balanced")`를 사용합니다.
  - 예측 시 `tech_score`, `prediction_confidence`, `is_uncertain`를 함께 산출해 신뢰도 기반 후처리를 가능하게 했습니다.

- **임베딩 유사도 기반 카테고리 분류 (Semantic Matching)**
  - 기술 카테고리 정의문 자체를 임베딩한 뒤, 기사 임베딩과의 `cosine similarity`로 top-1 / top-2를 구합니다.
  - `top1 score`와 `score gap` 임계치를 적용해 저신뢰 케이스를 `Other Tech`로 보냅니다.

- **규칙 기반 Stack/Entity 태깅 + 문맥 보정 (Hybrid 접근)**
  - 기술 스택 alias 사전(동의어/표기 변형)을 정규식으로 매칭합니다.
  - 모호한 용어(예: Go, Spark, Express)는
    - 문맥 키워드,
    - 지원 엔티티,
    - 부정 키워드
    를 반영해 점수화하여 오탐을 줄입니다.
  - 최종적으로 `stack_labels`, `primary_stack` 등 운영 지표 컬럼을 생성합니다.

- **트렌드 분석 피처 엔지니어링**
  - 카테고리/스택별 점유율(share), 성장률(growth), 소스 편향(lift), 신흥 키워드(emerging keywords) 등을 계산합니다.
  - 단순 건수 집계에서 더 나아가 해석 가능한 분석 지표를 생성합니다.

## 2) 최적화 및 속도 단축을 위해 도입된 개념

- **임베딩 캐시 (재학습/재실행 비용 절감)**
  - AG 학습 파이프라인에서 임베딩을 `.npy`로 저장/재사용해 반복 실험 시 인코딩 시간을 단축합니다.

- **모델/리소스 로딩 캐시 (`lru_cache`)**
  - 임베딩 모델, 카테고리 임베딩, alias 정규식 등을 한 번만 로딩/컴파일해 재사용합니다.

- **텍스트 길이 제한으로 계산량 절감**
  - 본문(`content`)과 최종 입력 텍스트(`text`) 길이를 상한으로 제한해
    - 임베딩 처리 시간,
    - 메모리 사용량,
    - 노이즈
    를 함께 줄입니다.

- **중복 제거 후 유니크 텍스트만 임베딩 (Dedup Encoding)**
  - URL/제목/텍스트 기준 중복을 먼저 제거합니다.
  - 분류 시에도 `factorize`로 중복 텍스트를 유니크 단위로만 임베딩하고, 결과를 원본 행에 복원합니다.

- **배치 처리 + 청크 연산**
  - 임베딩은 batch 단위로 인코딩합니다.
  - 카테고리 유사도 계산은 `SIMILARITY_CHUNK_SIZE` 기반 청크 처리로 대량 데이터에서도 메모리 급증을 방지합니다.

- **수치 타입 축소 (`float32`)**
  - 임베딩/유사도 계산 시 dtype을 `float32`로 맞춰 메모리 사용량과 계산 부담을 줄입니다.

- **통합 정규식(Combined Regex) 매칭**
  - stack alias를 하나의 결합 정규식으로 컴파일해 텍스트 스캔 횟수를 줄입니다.

- **품질 게이트를 통한 후속 연산량 절감**
  - 너무 짧은 기사, 저신호 제목, 과도한 재전송성 콘텐츠, 불확실 예측 등을 초기에 걸러
    이후 임베딩/분석 단계로 넘어가는 데이터량을 줄입니다.

---

## 3) 요약

이 프로젝트는 **임베딩+분류기+규칙기반 엔티티 추출의 하이브리드 파이프라인**을 사용합니다. 
속도 측면에서는 **캐시, 중복 제거, 배치/청크 처리, 입력 길이 제어, dtype 최적화**를 핵심 전략으로 사용해
정확도와 처리 비용 사이 균형을 맞추고 있습니다.
